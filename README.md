# AXI4 UVM Verification IP – Hoang Ho Extension

This branch extends Huy Le's self-contained AXI4 VIP-to-VIP UVM project. The goal is functional correctness for the main AXI4 memory-mapped. It is not a commercial compliance VIP.

## Original project by Huy Le

Huy Le's project provides:

- active master and active slave agents connected through `axi4_if`;
- AW, W, B, AR, and R channel handling;
- FIXED, INCR, and WRAP bursts;
- outstanding transactions and different-ID out-of-order responses;
- WSTRB, exclusive access, error responses, reset, backpressure, SVA, scoreboard, and functional coverage;
- QuestaSim Makefile and a reusable UVM test library.

The original class names, package structure, interface, and test flow are preserved.

## Hoang Ho additions

The extension focuses on functional corner cases required by Arm AXI4:

- corrected byte-lane handling for narrow and unaligned first transfers;
- corrected FIXED, INCR, WRAP address calculation and 4KB boundary checking;
- fixed duplicated W beats when `WREADY` remains HIGH;
- added per-`RID` read-beat reception and monitor reconstruction;
- preserved same-ID response order and different-ID out-of-order behavior;
- strengthened B/R dependency, WSTRB, EXOKAY, pending-transaction, and response-backpressure checks;
- added directed helper, 4KB, specification-corner, and response-backpressure tests;
- added `make spec_smoke`, fail-safe regression status, and safer coverage merging.

Existing Huy Le code remains unmarked. Hoang Ho changes use a short `// Hoang Ho:` comment. Entirely new SystemVerilog files begin with `// Hoang Ho - new file`.

## Main tests

Huy Le tests remain in the regression. Hoang Ho added:

- `axi4_4kb_boundary_test`
- `axi4_helper_unit_test`
- `axi4_spec_corner_test`
- `axi4_response_backpressure_test`

Run:

```bash
cd sim
make clean
make spec_smoke
make -j8 regress NUM_RUNS=5
make cov_report
```

Focused tests have been run on QuestaSim 10.6b. Final branch sign-off still requires the complete regression and coverage review after the latest commit.

## Limits

- No USER signals, ACE/coherency, AXI5 atomic features, or AXI-Stream.
- `AxCACHE`, `AxPROT`, `AxQOS`, and `AxREGION` are transported and checked, but no full system-level semantics are modeled.
- The default slave sends one complete R burst at a time; beat-level cross-ID interleaving is accepted by receivers but is not generated.
- Same-address concurrent read/write visibility uses a simple learning-model policy.
- Wider data-bus regression is outside the current scope.

The concise verification plan is in [`doc/AXI4_FULL_VPLAN.md`](doc/AXI4_FULL_VPLAN.md).
