# AXI4 Full Learning VIP Verification Plan

## 1. Objective and profile

This VPlan covers the self-contained VIP-to-VIP environment derived from Huy Le's AXI4-VIP. The target is functional correctness for the main AMBA AXI4 memory-mapped rules, not commercial VIP certification. The reference specification is Arm AMBA AXI and ACE Protocol Specification, Part A, with emphasis on Chapters A2-A8.

The default interface profile is 32-bit address, 32-bit data, and 4-bit transaction ID. Optional attributes (`AxCACHE`, `AxPROT`, `AxQOS`, `AxREGION`) are transported, held stable, compared, and covered, but their system-level cache, security, arbitration, and region semantics are not modeled.

## 3. Declared simplifications and waivers

These are deliberate learning-profile limits, not hidden compliance claims:

- No `*USER` signals.
- No ACE, coherency, AXI5 atomic transactions, parity, poison, MPAM, or AXI-Stream.
- No real cache, security permission, QoS arbitration, or REGION decode behavior.
- The default slave does not generate beat-level R interleaving, although receivers accept it.
- The slave is a simple byte-addressable memory/peripheral model, not an interconnect.
- The scoreboard checks completed transactions. Mixed read/write accesses to the same address are serialized in data-integrity tests to avoid imposing an unspecified system memory-ordering policy.
- Little-endian byte placement is used. Mixed-endian conversion is not modeled.

## 5. Coverage goals

Functional coverage is the primary closure metric. Code coverage from UVM factory macros and defensive error paths is excluded from protocol closure.

| Metric | Learning-VIP target |
|---|---:|
| Positive tests | 100% pass, no unexpected UVM_ERROR/UVM_FATAL |
| Core covergroups | >= 90%, with no unreviewed core bin misses |
| Assertions/directives | 100% exercised for enabled positive properties |
| Burst type x length x size | all legal target bins hit or justified |
| Alignment x size x burst | aligned, narrow, unaligned-first-beat, WRAP edge hit |
| Response x direction | OKAY, EXOKAY, SLVERR, DECERR hit as applicable |
| Timing | VALID-before-READY, READY-before-VALID, same-cycle, all five stalls |
| ID/outstanding | single ID, same-ID multiple outstanding, different-ID OOO |

## 6. Sign-off gate

Run the following in QuestaSim:

```bash
cd sim
make clean
make spec_smoke
make -j8 regress NUM_RUNS=5
make cov_report
```

Sign off the learning profile only when compilation is clean, `spec_smoke` passes, the complete positive regression exits with status 0, no pending monitor/scoreboard transactions remain, and the coverage report is reviewed against this VPlan. Static review alone is not simulation sign-off.
